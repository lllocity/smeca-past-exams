# Gemini インポート用プロンプト（確定版）

> このプロンプトを Gemini に貼り付け、続けて過去問 PDF を添付して使用する。
> 出力された JSON をそのまま `POST /api/import` に送信できる。

---

添付された中小企業診断士の過去問PDFを解析し、バックエンドのPostgreSQLデータベースにそのまま一括インポート（バルクインサート）できる、正確なJSON配列形式に変換してください。

### 【データ構造（JSONスキーマ）】
出力するJSON内の各オブジェクトは、必ず以下のキーとデータ型を厳守してください。

- `"subject_code"`: (文字列) 科目コード。以下のいずれかから適切に選択してください。
    - `"ECO"` (経済学・経済政策)
    - `"FIN"` (財務・会計)
    - `"MGT"` (企業経営理論)
    - `"OPS"` (運営管理)
    - `"LAW"` (経営法務)
    - `"MIS"` (経営情報システム)
    - `"SME"` (中小企業経営・中小企業政策)
- `"year"`: (数値) 試験の実施西暦年度（例: 2024）
- `"question_number"`: (数値) 問題番号。
    - 単独問（設問なし）: 整数のまま（例: `1`）
    - 大問＋設問構造の場合: `大問番号.設問番号`（例: 第10問の設問1 → `10.1`、設問2 → `10.2`）
- `"points"`: (数値) その問題の配点（例: 4）
- `"question_text"`: (文字列) 問題文。改行は `\n` で表現し、太字や箇条書きはMarkdown記法（`**`, `-` など）を使用すること。問題文や選択肢の中に、複雑な図表・グラフ・回路図・複雑な計算表が含まれており、テキストだけでの再現が不可能な場合は、`question_text` の末尾に必ず `\n\n*(※図表は管理画面からスクショを追加予定)*` と書き加えること。
- `"options"`: (配列) 選択肢のオブジェクト配列。構造は `[{"label": "ア", "text": "選択肢文"}, ...]` とすること。選択肢文の中で正誤を列挙する際は、全角コロン・全角読点を使った書式（例：`ア：誤、イ：正、ウ：誤、エ：誤`）で統一すること。
- `"correct_answer"`: (文字列) 正解のラベル（例: `"ア"`）。試験実施機関が全員正解とした問題は `"全員正解"` と記載すること。**必ず添付PDFの正解表・解答欄の該当行をそのまま転記すること。問題を自分で解いて判断したり、解説文から推定したりしないこと。正解が明記されていない場合は `null` とすること。**
- `"explanation"`: (文字列) 詳細な解説文。Markdown記法を用いて見やすく整形すること。各選択肢の正誤を列挙する際は、全角コロン・全角読点を使った書式（例：`ア：誤、イ：正、ウ：誤、エ：誤`）で統一すること。
- `"tags"`: (文字列の配列) その問題のテーマや論点を表すキーワードの配列（例: `["データベース", "正規化"]`）

### 【制約・出力ルール】
1. 前置きや「お役に立てて光栄です」などの余計なチャットテキストは一切省き、`[` から始まる純粋なJSONのみを出力してください。また、出力したJSONはコードブロック内に記載し、そのままファイルとしてダウンロードできる形式にしてください。
2. 問題文・選択肢・解説が途中で途切れないよう、全テキストを網羅してください。
3. `tags` は具体的な論点キーワードを2〜5個程度指定し、科目をまたいでも共通する概念（例: `"正規化"`, `"CVP分析"`）は統一した表記で記載してください。
